# Review

Run this at the end of a long work session to extract insights while the full session context is still available. This review examines two dimensions: what was learned about the project, and what was learned about the planner system itself.

## Instructions

Work through each section below. Produce concrete updates, not just observations. Every observation should result in an update to at least one planner artifact.

## Session Summary

What was accomplished this session? Brief narrative.

## Project Observations

Answer each question. Skip those that don't apply.

- What did this session reveal about the project?
- Were there testing gaps, architectural concerns, or recurring friction?
- Were decisions made implicitly that should be recorded in `decisions.md`?
- Did any findings warrant new `risks.md` entries or `backlog.md` items?
- Are there patterns worth codifying into a new guide?

## Planner Observations

Answer each question. Skip those that don't apply.

- Is the planner system itself working well?
- Are templates missing sections that were needed during this session?
- Did the runbook miss steps or handle edge cases poorly?
- Are any artifacts getting stale, redundant, or misused?
- Should naming conventions or folder structure change?
